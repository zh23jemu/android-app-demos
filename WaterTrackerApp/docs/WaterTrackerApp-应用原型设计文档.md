# WaterTrackerApp 应用原型设计文档

## 1. 项目概述

### 1.1 项目名称
WaterTrackerApp 喝水健康记录系统

### 1.2 项目背景
为了帮助用户养成规律饮水的健康习惯，设计并开发一款 Android 平台的本地喝水记录应用。应用支持饮水记录、每日目标设置、进度统计、历史查询以及达标庆祝特效，适合作为课程作业项目进行展示与答辩。

### 1.3 项目目标
- 实现快捷记录和自定义饮水量记录
- 支持设置并保存每日饮水目标
- 展示今日饮水统计和完成进度
- 展示今日明细与历史记录
- 达成目标后触发炫酷庆祝特效，增强交互体验

## 2. 用户需求分析

### 2.1 目标用户
- 学生：管理日常饮水习惯
- 普通用户：进行健康自我管理

### 2.2 核心需求
- 用户能够快速记录饮水量
- 用户能够查看今日已喝多少水
- 用户能够设置适合自己的每日目标
- 用户能够查看当天详细记录和历史数据
- 用户在完成目标时能够获得明显的反馈

## 3. 功能原型设计

### 3.1 功能模块划分
WaterTrackerApp 共设计 4 个主要功能模块：

1. 饮水记录模块
2. 目标设置与统计模块
3. 今日明细与历史查询模块
4. 达标反馈特效模块

其中前 3 个模块满足“至少 3 个核心功能模块”的课程要求，第 4 个模块作为视觉增强功能。

### 3.2 首页原型说明

首页主要由以下区域组成：

1. 顶部标题栏
- 显示应用名称“喝水健康记录”

2. 今日统计卡片区域
- 今日饮水总量
- 当前目标值
- 今日记录次数
- 进度条
- 达标状态文本
- 最近一次记录时间

3. 快捷记录区域
- `+250ml` 按钮
- `+500ml` 按钮
- 自定义毫升输入框
- 添加按钮

4. 目标设置区域
- 每日目标输入框
- 保存目标按钮

5. 今日明细区域
- 按时间展示今日每次饮水记录

6. 历史记录区域
- 按天展示历史饮水总量与记录次数

7. 达标庆祝特效层
- 当天第一次达到目标后显示粒子庆祝动画

### 3.3 交互原型说明

#### 快捷记录饮水
- 用户点击 `+250ml` 或 `+500ml`
- 系统立即记录并刷新统计信息

#### 自定义记录饮水
- 用户输入自定义毫升数
- 点击“添加”按钮后完成记录

#### 设置每日目标
- 用户输入新的目标毫升数
- 点击“保存目标”按钮
- 系统保存目标并重新计算进度

#### 查看今日明细
- 用户在首页可直接查看当天每次饮水的时间和毫升数

#### 达标庆祝
- 当天饮水总量首次达到目标
- 系统播放全屏粒子动画和达标文字提示

## 4. 页面流程设计

### 4.1 页面结构
WaterTrackerApp 采用单页面主界面结构，所有核心功能都集中在首页完成。

### 4.2 交互流程

#### 4.2.1 快捷记录流程
1. 用户点击快捷按钮
2. 系统插入一条饮水记录
3. 首页总量、进度、明细和历史记录同步刷新

#### 4.2.2 自定义记录流程
1. 用户输入自定义毫升数
2. 点击添加按钮
3. 系统校验输入是否合法
4. 保存记录并更新页面

#### 4.2.3 修改目标流程
1. 用户输入新的每日目标
2. 点击保存目标
3. 系统保存到本地配置
4. 根据新目标刷新进度条与状态文本

#### 4.2.4 查看历史流程
1. 系统读取数据库中的历史记录
2. 按日期聚合总饮水量和记录次数
3. 在历史区域列表中显示

#### 4.2.5 达标特效流程
1. 用户继续记录饮水
2. 系统判断今日总量是否首次达到目标
3. 若满足条件，则触发庆祝动画
4. 同一天不重复触发

## 5. 界面设计说明

### 5.1 设计原则
- 操作直接：一键记录和目标设置都在首页完成
- 数据清晰：总量、进度、明细、历史层次明确
- 展示友好：适合作业演示和截图
- 反馈明显：达标后提供强视觉反馈

### 5.2 色彩风格
- 主色调采用蓝绿色系，突出健康和清爽感
- 达标特效使用亮色粒子和白色文字，增强视觉冲击

### 5.3 可用性设计
- 输入框支持数字输入
- 错误输入时有明显提示
- 页面支持滚动，避免键盘遮挡输入框

## 6. 数据原型设计

### 6.1 饮水记录数据字段
- `id`：记录编号
- `date`：日期
- `amountMl`：饮水量
- `createdAt`：记录时间戳

### 6.2 历史汇总字段
- `date`：日期
- `totalMl`：当日总饮水量
- `recordCount`：当日记录次数

### 6.3 本地配置字段
- `daily_goal`：用户设置的每日目标值
- `last_celebration_date`：上一次触发达标特效的日期

## 7. 预期效果

- 用户能够方便快捷地记录饮水
- 用户能够查看今日饮水情况和历史趋势
- 用户能够自定义健康目标
- 达标时可获得更强的成就感和互动体验

## 8. 总结

WaterTrackerApp 的原型设计围绕“饮水记录、目标管理、数据展示、达标反馈”展开，结构清晰、功能完整、视觉效果突出，既满足课程作业对核心模块数量的要求，也具备较好的实用性和展示性。
