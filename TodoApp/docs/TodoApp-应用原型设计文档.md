# TodoApp 应用原型设计文档

## 1. 项目概述

### 1.1 项目名称
TodoApp 待办事项管理系统

### 1.2 项目背景
为了帮助用户高效管理日常任务、学习计划和生活事务，设计并开发一款轻量级本地待办事项管理应用。应用面向 Android 平台，强调操作简单、界面清晰、数据不丢失，适合作为课程作业项目进行展示与答辩。

### 1.3 项目目标
- 实现任务的新增、编辑、删除和完成状态切换
- 实现任务的本地持久化存储
- 实现任务状态与分类的筛选查看
- 展示任务统计信息，提升应用完整度和可视化效果

## 2. 用户需求分析

### 2.1 目标用户
- 学生：管理学习计划、作业安排
- 普通用户：管理生活待办、个人事项

### 2.2 核心需求
- 用户能够快速添加任务
- 用户能够查看并修改已有任务
- 用户能够按完成状态或分类筛选任务
- 用户关闭应用后再次打开，任务数据仍然存在
- 用户能够看到当前任务完成情况统计

## 3. 功能原型设计

### 3.1 功能模块划分
TodoApp 共设计 4 个主要功能区域：

1. 任务管理模块
2. 数据存储模块
3. 任务筛选模块
4. 任务统计模块

其中前 3 个模块满足“至少 3 个核心功能模块”的课程要求，第 4 个模块作为补强与展示增强。

### 3.2 首页原型说明

首页由以下区域组成：

1. 顶部标题栏
- 显示应用名称“待办事项”

2. 筛选区域
- 状态筛选：全部任务、未完成、已完成
- 分类筛选：全部分类、学习、生活
- 点击“筛选”按钮后刷新列表

3. 统计区域
- 总任务数
- 已完成任务数
- 未完成任务数

4. 任务列表区域
- 以列表形式显示所有符合条件的任务
- 每条任务显示标题、内容、分类、创建时间
- 每条任务支持完成勾选、编辑、删除

5. 浮动添加按钮
- 点击后弹出新增任务对话框

### 3.3 新增/编辑任务原型说明

弹窗中包含：
- 任务标题输入框
- 任务内容输入框
- 任务分类下拉框
- “取消”按钮
- “保存”按钮

交互逻辑：
- 标题为空时不允许保存
- 保存后自动刷新任务列表与统计数据
- 编辑任务时默认回填原有数据

## 4. 页面流程设计

### 4.1 页面结构
TodoApp 采用单页面主界面结构，通过弹窗完成新增与编辑操作。

### 4.2 交互流程

#### 4.2.1 新增任务流程
1. 用户点击右下角添加按钮
2. 系统弹出新增任务对话框
3. 用户输入标题、内容并选择分类
4. 用户点击保存
5. 系统将数据写入本地数据库
6. 首页列表和统计信息刷新

#### 4.2.2 编辑任务流程
1. 用户点击某条任务右侧编辑按钮
2. 系统弹出编辑任务对话框并回填原数据
3. 用户修改信息并保存
4. 系统更新数据库记录
5. 首页列表和统计信息刷新

#### 4.2.3 完成任务流程
1. 用户点击任务前方复选框
2. 系统更新该任务完成状态
3. 列表和统计卡片同步刷新

#### 4.2.4 删除任务流程
1. 用户点击删除按钮
2. 系统删除该任务
3. 列表和统计信息同步刷新

#### 4.2.5 筛选任务流程
1. 用户选择状态筛选条件
2. 用户选择分类筛选条件
3. 点击筛选按钮
4. 系统只显示符合条件的任务列表

## 5. 界面设计说明

### 5.1 设计原则
- 简洁清晰：避免过多复杂页面
- 操作直接：主要功能尽量在首页完成
- 信息集中：筛选、统计、列表集中显示
- 展示友好：适合作业截图与答辩演示

### 5.2 色彩风格
- 主色调采用绿色系
- 任务卡片采用 Material 卡片风格
- 浮动按钮用于突出新增操作

### 5.3 可用性设计
- 输入框提示文字明确
- 空列表时显示引导文本
- 操作按钮位置固定，便于使用

## 6. 数据原型设计

### 6.1 任务数据字段
- `id`：任务编号
- `title`：任务标题
- `content`：任务内容
- `category`：任务分类
- `isCompleted`：是否完成
- `createdAt`：创建时间

### 6.2 分类设计
- 学习
- 生活

### 6.3 筛选设计
- 状态：全部、未完成、已完成
- 分类：全部、学习、生活

## 7. 预期效果

- 用户可快速完成日常任务记录与管理
- 数据可长期保存在本地
- 支持按不同条件查看任务
- 能直观看到当前任务完成进度

## 8. 总结

TodoApp 的原型设计围绕“任务记录、任务筛选、任务统计、本地保存”展开，结构简单、功能完整、交互清晰，既满足课程作业对核心模块数量的要求，也具备较好的演示效果和后续扩展空间。
